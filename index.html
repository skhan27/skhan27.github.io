Professional Experience

Software Developer

MindBridge Ai, Feb 2018 - Present

At MindBridge Ai I am part of the Insights team which is responsible for data visualizations, dashboards and reports. The frontend of the core app involves working with AngularJS and Angular (hybrid application), Sass, Bootstrap and TypeScript. The REST service work involves working with a Java Spring application and writing database queries for MongoDB and PostgreSQL. The work that my team does includes the database queries to fetch/aggregate the data, the REST service work to handle shaping the data and the work on the frontend to render the dashboards. I built dynamic dashboards that aggregated over 1M rows of data from PostgreSQL and MongoDB databases and using Highcharts for rendering graphs. I also built a data caching system using MongoDB to prevent long running aggregations from hindering the user's experience. I was also part of the team to make the initial custom grid solution that was used across the app. Later I led the project to convert tables across the application to use ag-Grid. I was also responsible for improving the report generation infrastructure in Java to stream data from the database and allow creation of excel exports of more than 10M rows.

Biotechnology Lab Techician

McMaster University, May - December 2015

My main duties in this position were to research and design new experiments for use in course labs. I was also responsible for optimizing existing experiments using data from repeated experiments and writing/editing lab manuals. Lastly, I was also responsible for the preparation of chemicals and growth media for various courses.

Food Technologist

Olivieri Foods, January - August 2014

My main research project at Olivieri was to improve the flavor shelf life of their product. I was responsible for contacting various flavor companies to communicate our needs (ex: product has to be 'natural', enhance X and Y flavors etc). Once I had samples I made a bench scale test to determine which product from approximately 12 would be the best for our product. The testing cut the number down to 3 which were then selected full scale testing. The selected improvements were used for products that went to market in fall 2015. Other duties included setting up sensories and analyzing sensory data.
